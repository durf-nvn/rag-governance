• Phase 1: Backend Auth Enforcement & Security Headers
  - Enforced JWT Bearer Token validation across backend API routes (/ask-policy, 
    /paper-trail, /users, document management).
  - Derived user identity and roles strictly from verified JWT claims.
  - Restricted CORS to explicit domain origins while enabling all HTTP methods including PATCH for notification endpoints.
  - Added Security Headers middleware (X-Frame-Options: DENY, X-Content-Type-Options: 
    nosniff, X-XSS-Protection, Referrer-Policy).
  - Removed static fallback secret keys from utils.py.

• Phase 2: HttpOnly Cookie Migration & Storage Hardening
  - Backend /login now issues HttpOnly, SameSite=Lax authentication cookies.
  - Added /logout backend endpoint to invalidate auth cookies.
  - Configured global Axios credentials (withCredentials = true) in src/main.tsx.
  - Removed raw JWT token storage from browser sessionStorage in LoginPage.tsx 
    and DashboardLayout.tsx.

• Phase 3: Rate Limiting & Anti-Brute-Force Protection
  - Implemented in-memory sliding-window RateLimiter (backend/rate_limiter.py).
  - Protected /login (max 5 attempts/min) against credential stuffing.
  - Protected /auth/send-otp (max 3 requests/5min) against OTP spamming.
  - Protected /ask-policy (max 15 queries/min) against automated scraping.
  - Returns HTTP 429 Too Many Requests with dynamic Retry-After headers.

• Phase 4: RAG Prompt Injection Defense & Input Sanitization
  - Created input sanitizer and regex signature scanner (backend/sanitizer.py).
  - Intercepts instruction overrides ("ignore previous rules"), developer prompt 
    exfiltration ("show system prompt"), role impersonation ("act as admin"), 
    and jailbreaks before LLM or vector execution.
  - Verified with automated test suite (test_sanitizer.py).
